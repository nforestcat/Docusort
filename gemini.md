# 프로젝트: DocuSort AI (문서 자동 분류 및 요약 자동화)

## 1. 프로젝트 개요
- 다양한 형식의 문서(PDF, DOCX 등)를 분석하여 내용별로 폴더를 자동 분류하고, 논문(Paper)의 경우 핵심 내용을 요약하여 별도 저장하는 파이썬 기반 자동화 툴입니다.

## 2. 핵심 기능 및 워크플로우
1. **문서 스캔:** `input/` 폴더 내의 새로운 파일을 감지합니다.
2. **내용 분석:** 파일의 텍스트를 추출하여 주제(기술, 금융, 일반, 논문 등)를 판별합니다.
3. **폴더 분류:** 판별된 주제에 따라 `output/classified/[주제]/` 폴더로 파일을 이동합니다.
4. **논문 특화 요약:** 
   - 파일이 논문으로 판별될 경우, 초록(Abstract)과 결론(Conclusion)을 중심으로 요약합니다.
   - **참고문헌(References) 섹션 이후의 텍스트는 분석에서 제외**하여 효율성을 높입니다.
   - 요약본은 `output/summaries/[파일명]_summary.md` 형태로 생성합니다.

## 3. 기술 스택 (Tech Stack)
- **Language:** Python 3.10+
- **PDF 처리:** `PyMuPDF` 대신 Gemini CLI의 파일 직접 처리 기능을 우선적으로 사용합니다. (브랜치 전용 규칙)
- **AI/LLM:** Gemini CLI 혹은 Gemini API (분류 및 요약 로직용)
- **환경 관리:** `python-dotenv`를 사용하여 API 키 관리

## 4. 폴더 구조 규칙
에이전트는 아래 구조를 유지하며 개발을 진행합니다.
- `input/`: 분류할 원본 문서가 들어오는 곳
- `output/classified/`: 내용별 분류 폴더가 생성되는 곳
- `output/summaries/`: 논문 요약본(Markdown)이 저장되는 곳
- `src/`: 실제 파이썬 소스 코드 (`classifier.py`, `summarizer.py`, `utils.py` 등)

## 5. 코딩 및 에이전트 규칙
- **확인 절차:** 파일을 이동하거나 삭제하는 로직을 작성할 때는 반드시 사용자에게 먼저 계획을 보고하고 승인을 받으세요.
- **예외 처리:** PDF에 텍스트가 없거나(이미지 전용) 암호가 걸린 경우에 대한 예외 처리를 반드시 포함하세요.
- **결과물 형식:** 모든 요약본은 한국어로 작성하며, `# 요약`, `## 핵심 내용`, `## 결론` 등의 마크다운 형식을 따릅니다.
- **로그 기록:** 작업 수행 내역을 `automation.log` 파일에 기록하는 기능을 포함하세요.
