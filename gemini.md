# 프로젝트: DocuSort AI (문서 자동 분류 및 요약 자동화)

## 1. 프로젝트 개요
- 다양한 형식의 문서(PDF, DOCX 등)를 분석하여 내용별로 폴더를 자동 분류하고, 논문(Paper)의 경우 핵심 내용을 요약하여 별도 저장하는 파이썬 기반 자동화 툴입니다.

## 2. 핵심 기능 및 워크플로우
1. **문서 스캔:** `input/` 폴더 내의 새로운 파일을 감지합니다.
2. **정밀 내용 분석 및 분류:** 
   - `pymupdf4llm`을 사용하여 문서 구조(마크다운)를 확보합니다.
   - 정확한 판별을 위해 문서의 **앞부분(7,000자)**과 **뒷부분(3,000자)**을 샘플링하여 Gemini에게 전달합니다.
   - [논문, 행정서식, 과제금융, 기술매뉴얼, 일반안내] 중 하나로 분류하여 `output/classified/[주제]/`로 이동합니다.
3. **논문 특화 요약 및 이름 변경:**
   - 논문으로 판별된 경우, 강화된 정규표현식 패턴을 통해 **참고문헌(References) 섹션 이후를 제거**하여 분석 효율을 높입니다.
   - 추출된 서지 정보를 바탕으로 파일명을 `[연도]_[저자]_[키워드].pdf` 형식으로 변경하여 `processed/` 폴더에 저장합니다.
   - 요약본은 한국어로 작성하며 `output/summaries/[새파일명]_summary.md` 형태로 생성합니다.

## 3. 기술 스택 (Tech Stack)
- **Language:** Python 3.10+
- **PDF 처리:** `pymupdf4llm` (구조화된 마크다운 추출용)
- **AI/LLM:** `google-genai` SDK (`gemini-3.1-flash-lite-preview` 모델 사용)
- **API 제한 관리:**
  - **RPM 15, TPM 250k, RPD 250** 제한을 준수합니다.
  - 안정성을 위해 작업 간 최소 **5~10초의 대기 시간(time.sleep)**을 유지합니다.

## 4. 폴더 구조 규칙
에이전트는 아래 구조를 유지하며 개발을 진행합니다.
- `input/`: 분류할 원본 문서가 들어오는 곳
- `output/classified/`: 내용별 분류 폴더가 생성되는 곳
- `output/summaries/`: 논문 요약본(Markdown)이 저장되는 곳
- `src/`: 실제 파이썬 소스 코드 (`classifier.py`, `summarizer.py`, `utils.py` 등)

## 5. 코딩 및 에이전트 규칙
- **확인 절차:** 파일을 이동하거나 삭제하는 로직을 작성할 때는 반드시 사용자에게 먼저 계획을 보고하고 승인을 받으세요.
- **예외 처리:** PDF에 텍스트가 없거나(이미지 전용) 암호가 걸린 경우에 대한 예외 처리를 반드시 포함하세요.      
- **결과물 형식:** 모든 요약본은 한국어로 작성하며, `# 요약`, `## 핵심 내용`, `## 결론` 등의 마크다운 형식을 따릅니다.
- **파일 저장 및 인코딩:** 윈도우 환경(메모장 등)에서의 한글 깨짐 방지를 위해 **모든 텍스트 기반 파일(마크다운, 로그, 설정 파일 등) 저장 시 반드시 `utf-8-sig` 인코딩을 사용**합니다. 파이썬 코드 내 `open()` 함수 사용 시 `encoding='utf-8-sig'` 옵션을 명시해야 합니다.
- **로그 기록:** 작업 수행 내역을 `automation.log` 파일에 기록하는 기능을 포함하세요.
